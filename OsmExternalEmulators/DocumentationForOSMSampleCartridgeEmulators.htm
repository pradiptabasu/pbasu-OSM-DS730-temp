Documentation for OSM Sample Cartridge Emulators

Overview

The emulators project contains code and resources that represent the different fulfillment systems used by the OSM Sample Cartridge. The emulators consist of message driven beans that listen on the fulfillment system request queues, generate a response message, and send the responses back on the fulfillment system response queues. The message-driven beans use XQuery scripts to simulate responses for each fulfillment system.

Installation

1.       Download and unzip the Emulators.zip file to a directory.

2.       Import the project into Design Studio

Build the project

1.       Make a copy of the hostname.properties file under the root OsmExternalEmulators directory. Name the copied file <hostname>.properties using your machine’s hostname.

2.       Update the parameter settings inside of <hostname>.properties with your environment settings.

3.       Show the Ant view

a.       Window -> Show View -> Ant -> Ant

4.       Add the OsmExternalEmulators build.xml

a.       From the Ant window right-mouse click and select Add Buildfiles…

b.      Select OsmExternalEmulators->build.xml

5.       Clean the project

a.       Double-click the clean target

6.       Build the project

a.       Double-click the build target

Setup Weblogic JMS resources

1.       Stop Weblogic if necessary

2.       Modify the <weblogic_domain>/config/jms/oms_jms_module-jms.xml file

a.       Append the contents of OsmExternalEmulators/add-to-oms_jms_module-jms.xml to the end of the file. This will create the request and response queues for the fulfillment systems

3.       Start Weblogic

Deploy the emulators

1.       Double-click the deploy_emulators target from the Ant window. This deploys an application called OSM_external_emulators into the server.

Undeploy the emulators

1.       Double-click the undeploy target. This undeploys the OSM_external_emulators application from the server
